# 租户管理页面优化设计规格

## 1. 背景与目标

当前租户管理页已经覆盖新增、编辑、MCP 配置、可信域名、同步、诊断和删除等完整能力，但信息层级与操作密度偏高：表格操作区过宽、关键状态不易扫描、长表单依赖居中弹窗、窄屏体验不足。

本次优化把页面整理为一套专业、克制的运维工作台，在不改变后端接口和业务行为的前提下，提高租户状态的可读性、常用操作的效率以及配置过程的稳定感。

## 2. 范围

### 包含

- 仅优化 `admin-ui/src/pages/Tenants.jsx` 对应的租户管理页面。
- 重构页面信息层级、筛选工具栏、表格操作入口和租户配置交互。
- 将新增/编辑租户从居中弹窗调整为右侧抽屉。
- 补充页面级加载、空状态、错误提示、窄屏适配和必要的可访问性细节。
- 在不修改业务语义的前提下复用现有 API、字段、校验和操作流程。

### 不包含

- 不调整登录页、全局页头或整体路由结构。
- 不修改后端 API、数据库结构、同步规则或鉴权逻辑。
- 不新增数据统计接口；顶部状态数据由当前租户列表在前端派生。
- 不将窄屏表格改造成卡片列表。
- 不重新设计 MCP 配置、可信域名验证等独立业务流程，只调整其入口组织。

## 3. 选定方向

采用“专业运维工作台”方向：低饱和背景、明确的状态色、紧凑但留有呼吸感的表格，以及承载长配置表单的右侧抽屉。

页面识别特征是每行左侧的数据模式色轨：

- `stored` 使用青蓝色，表示本地存储/缓存模式。
- `direct` 使用紫色，表示企微 API 直连、无缓存模式。

色轨、模式标签和顶部状态摘要保持同一语义，帮助使用者快速定位数据模式，同时避免把整行铺成高饱和背景。

## 4. 视觉系统

### 色彩

| 语义 | 色值 | 用途 |
| --- | --- | --- |
| Ink | `#17323D` | 主标题、重要文本 |
| Stored teal | `#167D95` | stored 模式色轨与标签 |
| Direct violet | `#7356A3` | direct 模式色轨与标签 |
| Canvas | `#EEF3F5` | 页面背景与弱分区 |
| Running green | `#2C8C68` | 正常运行状态 |

状态色只承担状态或模式含义。危险操作继续使用 Ant Design 的错误色，告警使用其警告色，避免扩展出相互竞争的装饰色。

### 字体与密度

- 主体沿用系统 UI 字体与 Ant Design 字号体系。
- 租户 ID、CorpID 等识别码在适合的位置使用等宽字体，并支持复制或完整值提示。
- 表格保持中等密度；按钮和菜单的点击区域不低于组件默认尺寸。
- 使用细边框、轻背景分区和间距建立层级，不使用大面积阴影卡片。

## 5. 页面结构

页面从上到下分为四个区域：

1. 页面标题与简短说明。
2. 统一状态栏。
3. 搜索与筛选工具栏。
4. 租户表格。

### 5.1 页面标题

- 标题保持“租户管理”。
- 副文案说明该页用于管理租户接入、数据模式与同步状态。
- “新增租户”为该区域唯一主按钮。

### 5.2 统一状态栏

状态栏是一个连续容器，不拆成四张悬浮统计卡。展示：

- 全部租户
- 正常运行
- 直连模式
- 需要关注

数据由当前已加载的租户列表派生。状态栏用于概览，不承担复杂报表功能；点击统计项不强制联动筛选，避免引入未确认的交互语义。

“正常运行”统计 `enabled` 为真的租户；“需要关注”统计已禁用或缺少应用凭据（`has_secret` 为假）的租户。两项均只使用现有列表接口已经返回的字段，不新增状态推断。

### 5.3 工具栏

工具栏包含：

- 搜索框：按租户名称、租户 ID、CorpID 进行本地模糊匹配。
- 数据模式筛选：全部、stored、direct。
- 状态筛选：全部、已启用、已禁用。
- 刷新按钮：复用当前刷新逻辑，并在加载期间体现禁用/旋转状态。

筛选结果为空时显示“无匹配租户”，并提供清空筛选入口；这与系统完全无租户时的首次使用空状态区分开。

## 6. 表格设计

### 6.1 核心列

保留并重新组织当前业务需要的核心信息：

- 租户：名称、租户 ID，行左侧显示模式色轨。
- 企业信息：CorpID 等当前关键标识。
- 数据模式：使用明确的 stored/direct 标签。
- 运行状态：状态点、文本及必要的更新时间/辅助说明。
- 同步策略：展示同步间隔和已启用模块的简要摘要。
- 操作：固定在右侧，压缩为“配置”和“更多”。

具体列宽以避免常用标识和状态被截断为原则；长 ID 可以省略展示，但必须通过 Tooltip 或复制交互访问完整内容。

### 6.2 操作映射

主操作“配置”打开租户配置抽屉。其余操作进入“更多”下拉菜单：

| 菜单项 | 对应现有行为 |
| --- | --- |
| MCP 配置 | 打开现有 MCP 配置流程 |
| 可信域名 | 打开现有域名配置/验证流程 |
| 同步 | 触发现有增量同步流程 |
| 全量回滚 | 触发现有全量回滚/重建流程 |
| 诊断 | 打开现有同步诊断流程 |
| 删除 | 复用现有二次确认后删除 |

危险操作放在菜单末尾并用分隔线隔开。删除保持二次确认，不因进入菜单而弱化风险提示。

对于 `direct` 租户，同步、全量回滚和诊断操作保持禁用；禁用项同时给出“直连模式实时调用企微 API，无需同步”的原因，不能只依赖灰色表达。

## 7. 新增与编辑抽屉

### 7.1 抽屉规格

- 从右侧打开，桌面端采用适合长表单的固定或响应式宽度。
- 窄屏下宽度接近全屏，但保留清晰的关闭入口。
- 标题明确区分“新增租户”和“配置租户”。
- 抽屉正文独立滚动，底部操作栏固定。
- 底部操作为“取消”和主按钮“保存”。

### 7.2 表单分组

沿用现有字段和校验，将表单整理为四个视觉分区：

1. 基本信息：租户名称、标识等。
2. 连接凭据：CorpID、Secret 等企微接入字段。
3. 数据与同步策略：stored/direct 以及现有同步配置。
4. 域名配置：现有可信域名相关字段或说明。

分区只改善扫描和理解，不改变字段提交结构。敏感字段继续使用密码输入形态，不能在列表、提示或日志中回显完整值。

### 7.3 表单状态

- 打开编辑抽屉时填充当前租户数据；打开新增抽屉时清空旧值。
- 提交期间禁用重复保存，并显示明确加载状态。
- 保存成功后关闭抽屉并刷新列表，沿用当前成功提示。
- 保存失败时保留用户输入，错误提示使用后端可安全展示的信息。
- 表单发生修改后，关闭按钮、遮罩点击和“取消”均先请求确认；未修改时直接关闭。
- 模式切换导致字段适用性变化时，沿用现有校验和提交规则，不静默删除用户已输入但仍可能需要的值。

## 8. 既有辅助弹窗

MCP 配置和可信域名验证继续使用独立弹窗，避免把多个复杂任务同时塞进主配置抽屉。它们的入口统一移动到“更多”菜单，但内部业务行为保持不变。

诊断、同步和全量回滚继续复用现有请求与反馈机制。菜单触发后应立刻提供进行中状态，避免用户重复操作。

## 9. 数据流与状态管理

- 页面初始化和刷新仍调用当前租户列表接口。
- 搜索、模式筛选、状态筛选均基于已加载数据在前端派生，不新增网络请求。
- 顶部统计基于完整租户列表计算，不随本地搜索结果缩减，以保持概览稳定。
- 表格展示筛选后的列表。
- 新增、更新、删除或同步成功后调用现有刷新流程，确保统计与列表一致。
- 各租户的进行中操作应按租户 ID 隔离，避免一个租户的操作让整张表失去交互能力；全局刷新可使用表格级加载状态。

## 10. 加载、空状态与错误

### 加载

- 首次加载显示表格区域骨架或 Ant Design Table 的清晰加载态。
- 刷新时保留当前数据，叠加轻量加载反馈，减少页面跳动。
- 单租户操作只锁定对应入口。

### 空状态

- 系统没有任何租户时，显示简洁说明和“新增租户”主行动按钮。
- 有租户但筛选无结果时，显示“无匹配租户”和“清空筛选”，不显示新增引导。

### 错误

- 列表加载失败时在表格区域提供错误信息和重试入口。
- 操作失败使用现有消息机制；只展示经过现有 API 层处理、适合前端显示的错误，不输出凭据或原始堆栈。

## 11. 响应式与可访问性

- 桌面宽屏展示完整核心列，操作列固定在右侧。
- 中等宽度优先隐藏辅助说明、更新时间等次要信息，再启用横向滚动。
- 窄屏继续使用表格，不切换卡片；保留租户、模式、状态和操作等关键列。
- 抽屉在窄屏接近全宽，固定底栏不遮挡最后一个表单项。
- 图标按钮提供文本标签或 Tooltip；不能仅靠颜色区分状态。
- 下拉菜单、抽屉和确认框保持键盘可达，焦点由 Ant Design 组件管理并在关闭后合理返回触发点。
- 所有表单字段保留可见标签和校验反馈。

## 12. 建议实现边界

预计只修改或新增以下前端文件：

- 修改 `admin-ui/src/pages/Tenants.jsx`：页面结构、状态派生、筛选、操作菜单和抽屉交互。
- 新增 `admin-ui/src/pages/Tenants.css`：租户页专属布局、模式色轨、响应式规则。

实现时如发现现有公共样式已经提供等价能力，应优先复用；除非构建或规范明确要求，不修改 `App.jsx`、`Login.jsx` 和后端文件。

## 13. 验收标准

- 租户页呈现统一状态栏、搜索筛选工具栏和紧凑的核心表格。
- 每行通过色轨和标签明确区分 stored/direct。
- “配置”打开右侧抽屉，完整覆盖现有新增/编辑字段和校验。
- 其余操作完整迁移到“更多”菜单，原有行为不丢失。
- direct 模式的同步、全量回滚和诊断操作禁用且可理解原因。
- 未保存表单关闭前有确认，提交中不会重复保存。
- 首次加载、刷新、无租户、无筛选结果和加载失败均有明确反馈。
- 桌面与窄屏可用，窄屏表格允许横向滚动且抽屉接近全屏。
- 不修改后端契约、登录页和全局页头。
- `admin-ui` 构建通过，Ant Design 规范检查通过。
- 完成桌面与窄屏截图检查，并对新增、编辑、筛选、更多菜单、关闭确认执行交互冒烟测试。
