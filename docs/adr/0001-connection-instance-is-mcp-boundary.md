# ADR 0001：连接实例是唯一可管理的 MCP 边界

- 状态：Accepted
- 日期：2026-07-24

## 背景

平台曾同时提供连接级 `/mcp/{connection_id}` 与服务级
`/mcp/service/{service_id}`。服务层允许把多个连接的工具组合到一个端点，
但产品当前要求一个连接实例对应一个 MCP 端点。为每个连接自动创建默认服务
会形成 1:1 影子聚合，使 Token、工具策略、状态和控制台入口重复。

## 决策

连接实例成为租户唯一可管理的 MCP 能力边界：

- 新建连接时直接获得连接级 MCP 端点。
- MCP Token、工具策略、连接状态和调用日志均以连接实例为主。
- tenant 控制台和管理后台页面不再挂载独立 MCP 服务管理入口。
- 保留不进入 OpenAPI 的 admin 存量清理 API，只允许只读盘点、停用服务、揭示或撤销旧 Token；不能新建服务、修改绑定、签发 Token 或重新激活。
- 启动时不再为连接回填默认 MCP 服务。
- 已存在的 `/mcp/service/{service_id}` 可在兼容开关开启时继续运行，
  但只作为存量客户端和跨连接服务的兼容投影，不再是新业务入口。

## 后果

- 管理模型更简单，可信域名、附件验证、凭据、同步策略与 MCP 能力都在连接实例内完成。
- 存量服务 Token 不会被复制或重新解释为连接 Token。
- 跨连接服务不会被静默拆分或删除；其退出需要独立的数据盘点和迁移窗口。
- `mcp_service*` 表与服务网关暂时保留，待确认无存量依赖后再物理移除。

## 退出兼容层的条件

1. 生产环境不存在仍在调用的服务级端点。
2. 跨连接服务已拆分或由新的明确聚合产品承接。
3. 服务 Token 已撤销或过期。
4. 历史日志仍可按连接实例完成审计查询。
