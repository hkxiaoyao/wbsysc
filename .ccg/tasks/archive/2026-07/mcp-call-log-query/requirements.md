# MCP 调用日志查询需求

## 已确认

- 日志同时覆盖 MCP 业务工具调用与协议/鉴权访问。
- 管理页面需要区分“业务调用”和“协议访问”。
- 支持日志查询、检索和清理。
- 清理支持勾选删除、按当前筛选结果、按日期阈值和全部清空。
- 批量清理和全部清空必须二次确认，并明确展示影响范围。
- 自动清理保留天数可配置，默认 90 天，0 表示关闭；每天执行一次。
- 日志仅保存元数据、脱敏参数摘要、状态、错误码/安全错误摘要和耗时。
- 禁止保存 MCP Token、企微 Secret、完整请求正文和完整响应正文。

- 检索支持租户、日志类别、工具/事件、状态、时间范围、关键词、请求 ID、客户端 IP和耗时区间。
- 不提供高级查询表达式或原始 SQL 查询入口。
- 顶部导航增加“租户管理 / 调用日志”。
- 租户行操作菜单提供“查看调用日志”，进入日志页面并预筛选该租户。
- 租户日志视角顶部增加概览仪表盘。
- 全局与租户日志共用统一日志工作台；租户视角通过 URL 查询参数恢复和分享。
- 清除租户筛选后返回全局日志视角。
- 仪表盘展示调用总数、成功率、错误数、平均耗时、P95 耗时、调用趋势、工具排行和状态分布。
- 仪表盘统计随当前时间筛选联动。
- 新日志统一写入中心日志表；旧租户日志表保留兼容但停止写入。
- 上线迁移最近 90 天的旧租户日志到中心表。
- 迁移以来源 schema 和旧日志 ID 去重，允许安全重复执行。
