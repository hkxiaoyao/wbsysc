# 实施计划

## 目标态

连接实例是租户唯一可管理的 MCP 边界。现有服务层仅作为存量跨连接服务的运行时兼容层，不再提供新建或编辑入口，也不再为连接自动生成默认服务。

## Layer 1：领域与契约

1. 更新 `CONTEXT.md`，将 MCP 定义为连接实例的内建暴露能力。
2. 新增 ADR，记录取消独立服务聚合、保留存量兼容层的原因和退出条件。
3. 调整租户概览契约，移除服务聚合与服务绑定统计，以连接实例作为 MCP 实例统计来源。
4. 调整租户日志筛选契约，只暴露连接实例和源工具维度；历史日志字段继续保留在存储层。

## Layer 2：后端装配与兼容

1. `app/main.py` 不再挂载 tenant MCP 服务管理路由；仅保留隐藏的 admin 存量清理 API，用于只读盘点、停用、揭示和撤销旧 Token。
2. 停止启动时执行 `migrate_default_services`，避免继续产生 1:1 影子服务。
3. 保留受 `mcp_service_enabled` 控制的旧 `/mcp/service/{service_id}` 运行时，只用于存量客户端。
4. 保持 `/mcp/service` 路由优先级和关闭时 404 行为，避免被连接动态路由误捕获。

## Layer 3：前端收口

1. 租户控制台删除“服务”导航和页面入口，旧 `?view=services` 规范化到“连接”。
2. 管理后台删除“MCP 服务”导航和页面入口，旧链接规范化到“连接实例”。
3. 概览改为“连接实例/MCP 实例/调用日志”，不再展示独立服务。
4. 租户日志页移除服务 ID、工具别名筛选，只保留连接与源工具筛选。
5. 保留连接页面现有 MCP 地址、Token、工具策略、可信域名及校验文件上传功能。

## Layer 4：测试与交付

1. 先更新/新增契约测试，再修改实现。
2. 运行前端单测与生产构建。
3. 运行后端聚焦测试、Ruff/编译检查和完整 pytest。
4. 双模型审查 git diff；修复 Critical/Warning 后复验。
5. 更新任务记录、归档并提交。

## 明确不做

- 不物理删除 `mcp_service*` 表。
- 不把服务 Token 明文或密文复制为连接 Token。
- 不静默拆分跨连接服务。
- 不执行安全扫描。
