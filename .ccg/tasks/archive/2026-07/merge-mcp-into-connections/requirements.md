# 需求

## 目标

- 将 MCP 暴露能力融合到连接实例，一个连接实例对应一个 MCP 地址。
- 租户在连接实例内管理 MCP 令牌、工具策略和调用日志。
- 可信域名、附件上传/验证、凭据、数据和同步策略继续归连接实例/连接器管理。
- 移除用户可见的独立 MCP 服务管理概念。

## 兼容与安全边界

- 不删除或静默丢失存量跨连接服务绑定。
- 旧服务级 MCP 地址需要明确的兼容期或可验证迁移。
- 令牌迁移不能暴露明文或降低现有鉴权强度。
- 本任务不执行安全扫描；只运行常规单元、集成、类型和构建验证。

## 验收

- 租户控制台不再要求单独创建 MCP 服务。
- 每个连接实例可直接获得 MCP 地址并管理令牌、工具和日志。
- 连接实例已有配置与验证功能保持可用。
- 后端 API、数据库迁移、前端和测试形成闭环。
