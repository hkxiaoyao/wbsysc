# 实施前双模型分析

## 执行情况

- Gemini：已按要求调用，但本机未配置 `GEMINI_API_KEY`，后端退出且未产生分析报告。
- Claude：分析完成，覆盖设计规格、实施计划、现有 `Tenants.jsx` 与实际依赖版本。
- 实际锁定依赖：Ant Design `5.29.3`，不是 `package.json` 范围下限 `5.21.0`。

## 纳入实施的修正

- 加载中、加载失败、系统无租户、筛选无结果采用四种互斥状态，加载失败不能误导为“无租户”。
- direct 模式的禁用菜单项直接显示原因，不依赖禁用节点无法触发的 Tooltip。
- 行操作加载状态改为按 `tenant_id + action` 隔离，避免并发请求互相清除状态。
- Drawer 使用 Ant Design 5.29.3 的 `destroyOnHidden`，保存期间忽略关闭操作。
- 租户列不固定，窄屏只固定右侧操作列，降低双固定列遮挡风险。
- 保留可信域名的列表级扫描信息，并统一“全量回拨”术语。
- 搜索框补充可访问名称；视觉检查覆盖键盘操作与焦点返回。
- 校验命令使用仓库实际的 pnpm 工作流；Ant Design CLI 继续用于 API 查询，但不把其离线分析器误当成项目内 lint 依赖。

## 不纳入的建议

- 不改登录页、全局壳层、后端契约、同步语义和鉴权。
- 不引入卡片式移动布局或新的运行时依赖。
